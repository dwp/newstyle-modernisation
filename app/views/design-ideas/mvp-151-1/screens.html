Problem statement

We want to transform NSJSA by using the UC Build for managing and paying NSJSA claims. In the first MVP iteration, we don't want to change the NSJSA claimant experience - users will continue to apply through the existing service. To enable the back-end shift without impacting claimants, we need a reliable and efficient way for agents to transfer claim data from NSJSA Apply into the UC Build, using a proxy process that is based on UC phone claims without requiring direct claimant involvement.

Outcome statement

By enabling agents to transfer NSJSA claim data onto the UC platform using an adapted phone-claim proxy process, we aim to support payment and maintenance of a new NSJSA claim in a safe way with no impact on the current claimant experience.

This integration will help us gain insights on how we can pay claimants correctly and reliably. And it will lay the foundation for future claimant-facing improvements.

User needs
  1. As a Service centre agent,
    I need a way to transfer claim data from NSJSA onto the UC platform,
    so that I can manage NSJSA claims via the UC Build without impacting the claimant's experience.

  2. As a Service centre agent,
    I need to understand which data from NJSA is required by the UC platform and calculation tool,
    so that U can complete the proxy entry accurately and avoid delays.

  3. As a Service centre agent,
    I need the data entry process to mirror existing claim structures,
    so that I can efficiently transfer information without reinterpreting or reformatting it.

  4. As a Service centre agent,
    I need to track and manage NSJSA claims in the new system once they've been transferred,
    so that I can continue processing, updating and paying claims.
Hypothesis

We believe that by enabling agents to enter NSJSA claim data using a proxy entry process, will achieve that claims to be managed and paid via the UC service without impacting the claimant experience.

We will know this when we test a prototype and users tell us:

  • they can complete the data transfer efficiently without error
  • they can process claims seamlessly using UC and manual workarounds
  • they can support claimants with the same or improved service levels
